Role: Lead Frontend Architect & UI/UX Expert (React + Tailwind CSS + DaisyUI).

Context: The backend AI logic, Stress Test Lab, and memory systems are now fully functional. It is time to overhaul the frontend UI to match a premium, institutional-grade Quantitative Trading Platform. 

Task: Implement a viewport-locked layout, the "Cold Black" theme, and an interactive Equity Curve chart. DO NOT alter or break any existing backend API connections or background processes.

### 1. Structural Overhaul (Viewport & Layout Matrix)
- **Viewport Lock:** Set the main application wrapper to `h-screen w-screen overflow-hidden`. The page itself should never scroll.
- **Header:** A sleek, minimal top bar containing the app title, connection status badges, and a Theme Switcher dropdown.
- **Two-Column Workspace:**
  - **Left Column (approx 70-75% width):** The primary workspace containing the navigation tabs (Editor, Backtest, Hyperopt, Stress Test Lab, Results). This container must handle its own internal scrolling (`overflow-y-auto`).
  - **Right Column (approx 25-30% width):** The AI Chat Panel. This must be persistently docked to the right edge, spanning the full height below the header, with its own independent scroll. 

### 2. The "Cold Black" Theme Configuration (Tailwind / DaisyUI)
- Define a custom DaisyUI theme named `cold-black`.
- **Strict Color Rules:** ABSOLUTELY NO blue, slate, cyan, or navy tones. 
- **Backgrounds:** Use pure blacks and deep charcoal (`#000000`, `#09090b`, `#18181b`).
- **Text:** High contrast pristine white (`#fafafa`) for primary text, and muted silver/gray (`#a1a1aa`) for secondary text.
- **Borders/Dividers:** Sharp, thin, cold gray lines (`#27272a`).
- **Accents:** Use sharp Emerald Green for success/profitable signals and clear Amber/Soft Red for warnings or drawdowns.
- Ensure the theme is set as the default, but keep a toggle in the header if we add a Light mode later.

### 3. Cumulative Equity Curve Chart Integration
- In the **Results Tab**, implement a responsive `EquityCurveChart.jsx` component using `Recharts` or `Chart.js`.
- It must plot the cumulative profit line chronologically. 
- **Styling:** The line should be vibrant (e.g., Emerald Green) against the Cold Black background, with a soft gradient fill below the line fading into the dark background.
- Include an interactive tooltip that shows the specific Date, Trade Pair, and Cumulative Profit % when the user hovers over a data node.

Verify that all UI components (buttons, dropdowns, input fields) maintain WCAG AA contrast compliance against the new black backgrounds.
