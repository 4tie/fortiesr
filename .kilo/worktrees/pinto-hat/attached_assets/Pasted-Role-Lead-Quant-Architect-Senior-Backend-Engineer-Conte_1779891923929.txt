Role: Lead Quant Architect & Senior Backend Engineer.

Context: We are implementing three advanced institutional-grade risk management and meta-learning features into our Freqtrade Companion App. These features must operate autonomously in the backend to protect capital and improve the AI's long-term decision-making.

Task: Implement Market Regime Detection, a Dynamic Correlation Shield, and the AI Meta-Learning Ledger.

### 1. Market Regime Detection Filter (Built-in Strategy Template)
- **Objective:** Prevent trending strategies from taking trades in ranging markets, and vice-versa.
- **Action:** Update the autonomous strategy generation template tool (`generate_autonomous_strategy`).
- **Implementation:** Every generated strategy MUST now include a `Market Regime` block in `populate_indicators`.
  - Calculate `ADX` (Average Directional Index) and a fast/slow Moving Average (e.g., SMA 50 / SMA 200).
  - Add hyperoptable parameters in the companion `.json` file for `adx_threshold` (e.g., 20 or 25).
  - In `populate_entry_trend`, add a strict condition: if it's a trend-following strategy, it must enforce `dataframe['adx'] > self.adx_threshold.value`.
  - The AI must explicitly explain to the user in chat when it injects this filter to block ranging market chop.

### 2. Dynamic Pair Correlation Shield
- **Objective:** Prevent the AI from suggesting or backtesting a basket of highly correlated pairs (e.g., BTC, ETH, SOL all moving identically) to minimize systemic drawdown risk.
- **Action:** Create a new backend utility file: `backend/utils/correlation_engine.py`.
- **Implementation:**
  - Write a Python function `filter_uncorrelated_pairs(pairs_list, max_correlation=0.7)` that downloads the last 30 days of daily OHLCV data for the given pairs (mock this with Freqtrade's data provider or a simple CCXT call if data is missing).
  - Calculate a Pandas Pearson Correlation Matrix.
  - Return a mathematically diversified sub-list of pairs where no two pairs have a correlation higher than the threshold.
  - Expose this as an AI Tool: `analyze_pair_correlation`. The AI must run this tool before finalizing any multi-pair basket suggestion.

### 3. Meta-Learning Ledger (The "Lessons Learned" DB)
- **Objective:** Give the AI a persistent memory of failed code modifications and bad strategic concepts so it never repeats the same mistake across different chat sessions.
- **Action:** Create/manage `data/lessons_learned.json` in the root data folder.
- **Implementation:**
  - Update the `evaluate_plan` (or backtest evaluation) logic. If a newly tested strategy or modification results in a high drawdown (e.g., > 20%) or negative profit, trigger an automatic backend summary generator.
  - The AI must write a concise entry into `lessons_learned.json` (Format: `{"date": "...", "action": "...", "result_drawdown": "...", "lesson": "..."}`). Example: "Expanded RSI entry band on 5m timeframe for volatile pairs. Result: -12% profit due to fakeouts. Lesson: 5m timeframe needs volume confirmation, do not rely purely on oscillators."
  - **Crucial Context Injection:** Update the OpenRouter System Prompt generation in `ai_chat.py`. Read the last 10 lessons from `lessons_learned.json` and silently inject them into the system prompt under a `<CRITICAL_PAST_LESSONS>` tag. Instruct the AI: "Review these lessons. NEVER suggest a plan or write code that violates a lesson learned from past failures."
