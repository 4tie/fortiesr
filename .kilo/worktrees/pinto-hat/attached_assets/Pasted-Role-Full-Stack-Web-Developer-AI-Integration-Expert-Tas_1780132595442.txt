Role: Full-Stack Web Developer & AI Integration Expert.

Task: Implement the "Explain My Strategy" (AI Assistant) using a strictly local LLM (Ollama). The implementation MUST be fully dynamic, allowing the user to configure the Ollama API URL and dynamically select from downloaded models without any hardcoding.

### Phase 1: Backend Configuration & Endpoints (FastAPI)
1. **Config Management:** Update the application's settings manager to store two new configuration keys:
   - `ollama_api_url` (string, default: "http://localhost:11434")
   - `ollama_model` (string, default: "")
2. **Dynamic Models Fetcher:** Create `GET /api/ai/models`. This endpoint reads the current `ollama_api_url` from config, makes a GET request to `{ollama_api_url}/api/tags`, parses the JSON response, and returns a clean list of available model names (e.g., ["llama3:latest", "mistral:latest"]). Handle connection timeouts gracefully if the URL is unreachable.
3. **Explain Strategy Endpoint:** Create `POST /api/ai/explain-strategy` accepting a `strategy_name`. 
   - It reads the Python file of the strategy.
   - It fetches the `ollama_api_url` and `ollama_model` from the config.
   - It sends a request to `{ollama_api_url}/api/generate` with a system prompt: "You are a trading mentor. Explain this Freqtrade Python strategy logic in simple Arabic using real-world trading analogies. Be concise."
   - Return the generated response.

### Phase 2: Frontend Implementation - Settings Tab
1. **AI Settings Section:** In the existing Settings Tab/Page, add a new section for "AI Assistant Configuration".
2. **Connection Input:** Add a text input for the "Ollama API URL" (supporting localhost, Cloudflare tunnels, Tailscale IP, etc.).
3. **Dynamic Model Dropdown:** - Add a dropdown for "Select AI Model".
   - Create a "Refresh Models" button or trigger a fetch automatically when the API URL changes.
   - The dropdown should populate using the data from `GET /api/ai/models`.
4. **Save Config:** Ensure changes to the URL and Model are saved to the backend configuration.

### Phase 3: Frontend Implementation - The Assistant Trigger
1. **Trigger Button:** In the `PerformanceTab` (or Strategy selector), add a "🧠 أشرح لي" (Explain Logic) button.
2. **Display UI:** When clicked, show a loading state, then render the AI's explanation in a clean, scrollable Modal/Drawer using Markdown rendering.
3. **Error Handling:** If Ollama is unreachable (e.g., VPN off or wrong URL), display a clear toast notification directing the user to check their Settings.
