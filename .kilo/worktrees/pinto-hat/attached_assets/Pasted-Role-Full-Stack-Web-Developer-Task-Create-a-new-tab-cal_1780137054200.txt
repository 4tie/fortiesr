Role: Full-Stack Web Developer.

Task: Create a new tab called "Pair Explorer" in the application. This feature allows testing a single strategy against a list of multiple trading pairs simultaneously to find the best-performing assets.

### Phase 1: Backend Implementation (FastAPI)
1. Create a `POST /api/strategy/pair-explorer` endpoint.
2. The endpoint should accept: `strategy_name`, `list_of_pairs` (array), and `backtest_config` (timeframe, timerange, etc.).
3. Implementation Logic:
   - Since running many backtests sequentially is slow, implement a task queue or parallel execution (using `asyncio` or `concurrent.futures`) to run backtests on the provided pairs.
   - Respect the `max_allowed_trades` or resource concurrency limits to avoid crashing the machine.
   - For each pair, run the Freqtrade backtest command: `freqtrade backtest --strategy {strategy_name} --pairs {pair} ...`
   - Capture the result (Profit/Loss, Winrate, Sharpe ratio) for each pair.
4. Return a structured JSON response containing the results of all tested pairs.

### Phase 2: Frontend Implementation (React)
1. Create the "Pair Explorer" Tab UI:
   - Input: A text area for the user to paste a list of trading pairs (e.g., BTC/USDT, ETH/USDT, SOL/USDT).
   - Input: Strategy Selector (pre-selected from the active one).
   - Input: Backtest configuration (timerange, etc.).
   - Action Button: "Run Exploration" button.
2. Display: A table showing the results:
   - Columns: Pair Name | Total Profit % | Win Rate | Sharpe Ratio | Status (Completed/Failed).
   - Add sorting functionality (e.g., sort by Profit %).
3. Visual Indicator: A progress bar showing how many pairs are processed out of the total.

### Phase 3: Integration
1. Ensure this tab uses the same strategy files as the main Optimizer tab.
2. Add an "Apply to Strategy" button next to any successful pair in the results table, which saves that pair into the strategy configuration.

Please provide an execution plan for parallelizing the backtests to ensure system stability.
