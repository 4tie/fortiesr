Role: Full-Stack Web Developer & UI/UX Expert.

Task: Build a "Smart Trading Pair Selector" UI component for the Web Frontend, strictly utilizing the existing Python `PairSelectorService` logic. Do NOT rewrite the core selection logic; your job is to expose it via API and build the UI.

### Phase 1: API Routing (Backend)
Check if FastAPI routes exist for the `PairSelectorService`. If not, create a router (e.g., `backend/api/routers/pairs.py`) to expose these actions:
1. `GET /api/pairs/state` -> Returns the full state (available, selected, favorites, locked, max_trades).
2. `POST /api/pairs/toggle-favorite` -> Toggles star status.
3. `POST /api/pairs/toggle-lock` -> Toggles padlock status.
4. `POST /api/pairs/toggle-select` -> Selects/Deselects a pair.
5. `POST /api/pairs/randomize` -> Triggers the randomize function based on max open trades.
6. `POST /api/pairs/update-max-trades` -> Updates the limit.
7. `POST /api/pairs/clear` -> Clears all non-locked pairs.

### Phase 2: UI Component (`PairSelectorWidget.jsx` or similar)
Create a modern, dark-themed, scrollable UI widget with the following specifications:
1. **Header Controls:**
   - A search bar to easily find pairs.
   - A numeric input or small slider for "Max Open Trades".
   - A "Randomize" button (🎲) that triggers the backend randomize function (which respects the max trades and locked pairs).
   - A "Clear Selection" button.
2. **Pair List Items (The Rows):**
   - Display the list dynamically. Keep favorites pinned to the top or clearly identifiable.
   - Each row must have:
     - The Pair Name (e.g., BTC/USDT).
     - **Star Icon (⭐):** Clickable to toggle favorite status.
     - **Padlock Icon (🔒/🔓):** Clickable to lock the pair (visually indicating it won't be removed by clear/randomize).
     - **Checkbox/Switch:** To manually select or deselect the pair.
3. **UX Behavior:**
   - Ensure the UI feels snappy. When a user clicks a Star or Lock, send the API request in the background and refresh the state list so the UI reflects the true backend `pair_selector_state.json`.
