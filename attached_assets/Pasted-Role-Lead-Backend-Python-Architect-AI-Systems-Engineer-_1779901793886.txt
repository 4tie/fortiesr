### 1. The `@deep` Trigger (Chat Router)
- In the main `ai_chat.py` (or chat router endpoint), add logic to inspect the incoming user message.

### 2. The Multi-Agent Architecture (Backend Logic)
Create the `execute_deep_agentic_loop` function to orchestrate the following workflow:

- **Agent A (The Proposer):** Instructed to act as a Quant Developer. It takes the user's prompt, formulates a hypothesis, writes initial strategy code, or proposes parameter adjustments.
- **Agent B (The Validator/Critic):** Instructed to act as a strict Risk Manager. It receives Agent A's output. It MUST possess the ability to trigger tool calls (like running a backtest, checking syntax, or retrieving historical data). It analyzes Agent A's work and outputs a critique (e.g., "The code is structurally sound, but the risk-to-reward ratio is poor. Adjust the stoploss to X and retry").

### 3. The Finite Loop Implementation (Safety Constraints)
- Implement a `while` or `for` loop governing this interaction with a strict `MAX_ITERATIONS = 2` (or 3) to prevent infinite loops.
- **Iteration Flow:**
  1. Agent A generates Proposal 1.
  2. Agent B evaluates Proposal 1 (optionally running tools) and returns Critique 1.
  3. Agent A receives Critique 1 and generates Proposal 2 (Refinement).
  4. Agent B evaluates Proposal 2.
- **Loop Exit:** The loop exits when `MAX_ITERATIONS` is reached, OR if Agent B explicitly returns a "VALIDATION_SUCCESS" token indicating the proposal is perfect.

### 4. Output Aggregation
- Once the loop concludes, synthesize the final refined proposal from Agent A, along with a brief summary of the internal dialogue (e.g., "Deep Analysis complete: 2 iterations run. We found an issue with X and refined it to Y").
- Return this final synthesized payload to the frontend as the standard AI response.
