We inspected the Optimizer system. Implement only the next small safe step:

Add a safe "Promote Best Trial to Candidate" workflow.

Do not implement Smart Parameter Selection yet.
Do not implement AutoQuant handoff yet.
Do not refactor the optimizer.
Do not remove existing endpoints.
Do not change AutoQuant.
Do not modify accepted versions automatically.

Current finding:
The Optimizer currently stores trial params inside session.trials.
The dangerous existing endpoint POST /api/optimizer/apply-trial writes directly to the accepted version params.json.
VersionManager.create_version_from_optimizer_trial() already exists and can create a candidate version, but it is not wired to any router endpoint.
The UI currently has "Apply Trial #N" / "Apply & Export Best", which is dangerous because it modifies the accepted version directly.

Goal:
Add a safe candidate-promotion path so the best optimizer trial can be saved as a candidate version without touching the original strategy or accepted version.

Backend requirements:

1. Add endpoint:

POST /api/optimizer/session/{session_id}/best-trial/promote-candidate

Behavior:

* Load optimizer session by session_id.
* Validate session exists.
* Validate session is completed.
* Validate best_trial exists.
* Validate best_trial has parameters and successful metrics.
* Call VersionManager.create_version_from_optimizer_trial().
* Create a candidate version from the best trial.
* Do not modify accepted version.
* Do not modify original .py or original .json.
* Return:
  {
  "ok": true,
  "strategy_name": "...",
  "candidate_version_id": "...",
  "trial_number": 1,
  "score": ...,
  "metrics": {...}
  }

2. Add optional endpoint for any selected trial:

POST /api/optimizer/session/{session_id}/trial/{trial_number}/promote-candidate

Same behavior, but uses the selected trial instead of best_trial.

3. Add structured params endpoint:

GET /api/optimizer/session/{session_id}/best-trial/params

and optionally:

GET /api/optimizer/session/{session_id}/trial/{trial_number}/params

This should convert flat params like:

* buy__x
* sell__x
* roi__60
* stoploss__value
* trailing__stop
* trailing__positive
* trailing__offset

Into Freqtrade-compatible format:

{
"strategy_name": "...",
"params": {
"buy": {},
"sell": {},
"roi": {},
"stoploss": {},
"trailing": {}
}
}

Mapping rules:

* buy__x → params.buy.x
* sell__x → params.sell.x
* roi__60 → params.roi["60"]
* stoploss__value → params.stoploss.stoploss
* trailing__stop → params.trailing.trailing_stop
* trailing__positive → params.trailing.trailing_stop_positive
* trailing__offset → params.trailing.trailing_stop_positive_offset
* trailing__positive_offset → params.trailing.trailing_stop_positive_offset
* trailing__only_offset_is_reached → params.trailing.trailing_only_offset_is_reached

Preserve unknown namespaces safely if they exist, but do not invent unsupported Freqtrade fields.

Frontend requirements:

1. In OptimizerTab.jsx, after a session completes and has a best trial, add:

* View Best Params
* Promote Best Trial to Candidate

2. "View Best Params":

* Calls GET /api/optimizer/session/{session_id}/best-trial/params
* Shows structured JSON in a modal
* Has Copy JSON button

3. "Promote Best Trial to Candidate":

* Calls POST /api/optimizer/session/{session_id}/best-trial/promote-candidate
* Shows success message with candidate_version_id
* Does not call apply-trial
* Does not change accepted version

4. Existing dangerous button:
   Do not delete it yet, but rename it or mark it clearly:

* Rename "Apply & Export Best" to "Overwrite Accepted Params"
* Add a strong confirmation warning:
  "This will overwrite the accepted version params.json. Use Promote to Candidate for the safe workflow."

Safety requirements:

* Original strategy file must not change.
* Original params JSON must not change.
* Accepted version must not change.
* Candidate version must contain strategy.py and params.json.
* Existing optimizer run behavior must keep working.
* Existing apply-trial endpoint can remain for backward compatibility, but it must not be the primary UI action.

Acceptance criteria:

* Run optimizer normally.
* After completion, View Best Params returns structured Freqtrade JSON.
* Promote Best Trial to Candidate creates a candidate version.
* Accepted version is unchanged.
* Original strategy files are unchanged.
* UI shows candidate_version_id after success.
* Dangerous overwrite action is clearly labeled and requires confirmation.
